许多 PC 端表格组件拥有大量列字段。在移动端竖屏模式下，横向空间严重不足，用户需要频繁横向滑动才能查看完整数据。旋转到横屏方向可以有效地交换屏幕的宽高，为表格内容提供更宽的视口。

## 为什么选择 CSS `transform: rotate()` 方案？

实现移动端横屏有多种方式，但每种都有明显的局限性：

| 方案 | 缺陷 |
|------|------|
| **`screen.orientation.lock('landscape')`** | 需要页面先进入全屏模式（`Element.requestFullscreen()`）。大多数浏览器在非全屏或非用户手势触发时会拒绝锁定，且 iOS Safari 完全不支持此 API |
| **强制用户旋转设备** | 依赖用户的物理操作和设备的自动旋转设置，无法通过代码触发，且许多用户会锁定屏幕方向 |
| **响应式重新设计** | 需要为窄屏完全重新设计表格布局——成本高昂，且对于天然需要多列的数据密集型表格并不适用 |

**CSS `transform: rotate(90deg)`** 规避了上述所有问题：

- **全浏览器、全设备兼容** — 无 API 限制，无权限弹窗
- **纯代码可控** — 通过按钮点击触发，不依赖设备方向
- **保留原始表格布局** — 无需重新设计列结构，PC 端表格原样复用
- **零运行时开销** — GPU 合成层变换，不触发重排或重绘

唯一的副作用是 CSS 旋转后，触摸的物理轴向与元素的逻辑滚动轴向不再对齐。这正是 `v-mobile-table` 的 `rotation` 参数所解决的——它会自动对触摸坐标进行逆变换矫正，使滑动方向与旋转后的表格逻辑方向一致。

## 工作原理

1. 使用 CSS `transform: rotate(90deg)` 旋转表格容器
2. 交换宽高（`width: 100vh`，`height: 100vw`），使旋转后的容器填满屏幕
3. 在指令上设置 `rotation: 90`——内部 `transformCoords()` 会将触摸坐标 `(clientX, clientY)` 映射为 `(clientY, -clientX)`，从而使滚动方向与旋转后的表格逻辑轴对齐

## 配置说明

`rotation` 参数支持以下值：

| 值 | 说明 |
|----|------|
| `0` | 不旋转（默认值） |
| `90` | 容器顺时针旋转 90° |
| `-90` | 容器逆时针旋转 90° |
| `180` | 容器旋转 180° |

```vue
<template>
  <div v-mobile-table="{ preset: 'element-plus', rotation: 90 }">
    <el-table :data="tableData" height="100%">
      <!-- 表格列 -->
    </el-table>
  </div>
</template>
```

## 旋转定位原理

`transform-origin: top left` 配合 `translate: 100vw 0` 是关键的定位技巧。绕左上角顺时针旋转 90° 后，容器会偏移到屏幕左侧之外，而 `translate: 100vw 0` 将其推回，使其刚好覆盖整个视口。

## 注意事项

### 横屏时建议关闭边缘检测

横屏模式下，建议将 `disableEdgeDetection` 设为 `true`。主要原因是：在横向浏览宽表时，手势经常会滑到屏幕边缘；如果继续保留边缘检测，手势可能在边缘被释放并触发系统原生事件（例如返回手势）。关闭边缘检测后，边缘滑动会继续由表格滚动逻辑接管，避免误触发系统级操作。

```ts
const landscapeScrollOptions = {
  preset: 'element-plus' as const,
  rotation: 90,
  disableEdgeDetection: true,
}
```

如果你是通过状态切换横屏，也可以显式写成：

```ts
const scrollOptions = computed(() => ({
  preset: 'element-plus' as const,
  rotation: isLandscape.value ? 90 : 0,
  disableEdgeDetection: isLandscape.value ? true : false,
}))
```

### 安全区域适配

在带有刘海屏或圆角的设备（如 iPhone）上，建议将安全区应用到横屏容器本身：

```css
.landscape-container {
  padding-top: env(safe-area-inset-top);
  padding-right: env(safe-area-inset-right);
  padding-bottom: env(safe-area-inset-bottom);
  padding-left: env(safe-area-inset-left);
  box-sizing: border-box;
}
```

### 响应式尺寸

当你的项目使用框架自带的响应式字体缩放（例如字体大小会根据屏幕尺寸自动增大）时，旋转到横屏模式会触发这种缩放。但如果表格列宽是固定的像素值，它们不会按比例增大，导致列标题换行或内容溢出。

**典型场景：**
- 竖屏模式：font-size = 14px，列宽 = 100px（视觉平衡）
- 横屏模式：font-size = 18px（自动缩放），列宽 = 100px（未变化）→ 标题文本换行

**解决方案：** 使用一个辅助函数，根据视口宽度和最大宽度约束来动态计算列宽：

```typescript
// 使用中请删掉下面这行注释
// eslint-disable-next-line no-restricted-imports
import { round } from "lodash-es"

// 理想宽度，设计稿的宽度（移动端参考宽度）
const idealWidth = 375

// 表示伸缩视图的最大宽度
const maxWidth = 600

/**
 * 限制大小的 vw 转换
 * @param {number} n - 基础宽度（像素，基于理想宽度）
 * @returns {string} CSS min/max 函数字符串
 */
export function vw(n: number) {
  if (n === 0) return n

  const vwN = round((n * 100) / idealWidth, 3)
  const maxN = round((n * maxWidth) / idealWidth, 3)
  const cssF = n > 0 ? "min" : "max"
  return `${cssF}(${vwN}vw, ${maxN}px)`
}
```

在列定义中应用 `vw()` 函数：

```vue
<script setup lang="ts">
import { vw } from './utils/vw'

const columns = [
  { prop: 'id', label: 'ID', width: vw(60) },
  { prop: 'name', label: '姓名', width: vw(120) },
  { prop: 'email', label: '邮箱', width: vw(200) },
  { prop: 'phone', label: '电话', width: vw(150) },
]
</script>

<template>
  <div v-mobile-table="{ preset: 'element-plus', rotation: isLandscape ? 90 : 0 }">
    <el-table :data="tableData" height="100%">
      <el-table-column
        v-for="col in columns"
        :key="col.prop"
        :prop="col.prop"
        :label="col.label"
        :width="col.width"
      />
    </el-table>
  </div>
</template>
```

这种方法确保随着视口（和字体大小）增大，列宽按比例缩放，在所有屏幕尺寸上保持视觉平衡。

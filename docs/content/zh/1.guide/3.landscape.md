## 为什么需要横屏？

许多 PC 端表格组件拥有大量列字段。在移动端竖屏模式下，横向空间严重不足，用户需要频繁横向滑动才能查看完整数据。旋转到横屏方向可以有效地交换屏幕的宽高，为表格内容提供更宽的视口。

### 为什么选择 CSS `transform: rotate()` 方案？

实现移动端横屏有多种方式，但每种都有明显的局限性：

| 方案 | 缺陷 |
|------|------|
| **`screen.orientation.lock('landscape')`** | 需要页面先进入全屏模式（`Element.requestFullscreen()`）。大多数浏览器在非全屏或非用户手势触发时会拒绝锁定，且 iOS Safari 完全不支持此 API |
| **强制用户旋转设备** | 依赖用户的物理操作和设备的自动旋转设置，无法通过代码触发，且许多用户会锁定屏幕方向 |
| **响应式重新设计** | 需要为窄屏完全重新设计表格布局——成本高昂，且对于天然需要多列的数据密集型表格并不适用 |

**CSS `transform: rotate(90deg)`** 规避了上述所有问题：

- **全浏览器、全设备兼容** — 无 API 限制，无权限弹窗
- **纯代码可控** — 通过按钮点击触发，不依赖设备方向
- **保留原始表格布局** — 无需重新设计列结构，PC 端表格原样复用
- **零运行时开销** — GPU 合成层变换，不触发重排或重绘

唯一的副作用是 CSS 旋转后，触摸的物理轴向与元素的逻辑滚动轴向不再对齐。这正是 `v-table-touch-scroll` 的 `rotation` 参数所解决的——它会自动对触摸坐标进行逆变换矫正，使滑动方向与旋转后的表格逻辑方向一致。

## 工作原理

1. 使用 CSS `transform: rotate(90deg)` 旋转表格容器
2. 交换宽高（`width: 100vh`，`height: 100vw`），使旋转后的容器填满屏幕
3. 在指令上设置 `rotation: 90`——内部 `transformCoords()` 会将触摸坐标 `(clientX, clientY)` 映射为 `(clientY, -clientX)`，从而使滚动方向与旋转后的表格逻辑轴对齐

## 配置说明

`rotation` 参数支持以下值：

| 值 | 说明 |
|----|------|
| `0` | 不旋转（默认值） |
| `90` | 容器顺时针旋转 90° |
| `-90` | 容器逆时针旋转 90° |
| `180` | 容器旋转 180° |

```vue
<template>
  <div v-table-touch-scroll="{ preset: 'element-plus', rotation: 90 }">
    <el-table :data="tableData" height="100%">
      <!-- 表格列 -->
    </el-table>
  </div>
</template>
```

## 完整示例：按钮切换横屏

以下示例演示了通过按钮在竖屏和横屏模式之间切换，并动态更新 `rotation` 和 `disableEdgeDetection` 参数：

```vue
<script setup lang="ts">
import { computed, ref } from 'vue'
import { vTableTouchScroll } from 'vue-table-touch-scroll'

const isLandscape = ref(false)

const scrollOptions = computed(() => ({
  preset: 'element-plus' as const,
  rotation: isLandscape.value ? 90 : 0,
  disableEdgeDetection: isLandscape.value,
}))

const toggleLandscape = () => {
  isLandscape.value = !isLandscape.value
}

const tableData = Array.from({ length: 30 }, (_, i) => ({
  id: i + 1,
  name: `User ${i + 1}`,
  age: 20 + (i % 40),
  email: `user${i + 1}@example.com`,
  phone: `+1 ${String(i).padStart(3, '0')}-${String(i * 10).padStart(4, '0')}`,
  department: ['Engineering', 'Design', 'Marketing', 'Sales'][i % 4],
  address: `No. ${i + 1}, Street ${i % 10}, City ${i % 5}`,
  date: new Date(2024, i % 12, (i % 28) + 1).toLocaleDateString(),
}))
</script>

<template>
  <div>
    <button @click="toggleLandscape">
      {{ isLandscape ? '退出横屏' : '进入横屏' }}
    </button>

    <div
      v-table-touch-scroll="scrollOptions"
      :class="['table-container', { 'is-landscape': isLandscape }]"
    >
      <el-table :data="tableData" height="100%" border stripe>
        <el-table-column prop="id" label="ID" width="60" fixed="left" />
        <el-table-column prop="name" label="姓名" width="120" />
        <el-table-column prop="age" label="年龄" width="80" />
        <el-table-column prop="email" label="邮箱" width="280" />
        <el-table-column prop="phone" label="电话" width="180" />
        <el-table-column prop="department" label="部门" width="120" />
        <el-table-column prop="address" label="地址" width="280" />
        <el-table-column prop="date" label="日期" width="120" />
      </el-table>
    </div>
  </div>
</template>

<style scoped>
.table-container {
  width: 100%;
  height: 400px;
  transition: transform 0.3s ease;
}

.table-container.is-landscape {
  position: fixed;
  top: 0;
  left: 0;
  z-index: 1000;
  width: 100vh;
  height: 100vw;
  transform: rotate(90deg);
  transform-origin: top left;
  translate: 100vw 0;
}
</style>
```

## 注意事项

### 旋转定位原理

`transform-origin: top left` 配合 `translate: 100vw 0` 是关键的定位技巧。绕左上角顺时针旋转 90° 后，容器会偏移到屏幕左侧之外，而 `translate: 100vw 0` 将其推回，使其刚好覆盖整个视口。

### 过渡动画

在容器上添加 `transition: transform 0.3s ease` 可以在切换横竖屏时提供平滑的旋转动画效果。你还可以在过渡期间添加背景遮罩层以获得更好的用户体验。

### 安全区域适配

在带有刘海屏或圆角的设备（如 iPhone）上，需要注意安全区域的适配：

```css
.table-container.is-landscape {
  padding: env(safe-area-inset-top) env(safe-area-inset-right)
           env(safe-area-inset-bottom) env(safe-area-inset-left);
}
```

### 退出按钮设计

在横屏模式下，建议在固定位置放置一个浮动的"退出"按钮，方便用户快速返回竖屏模式：

```vue
<template>
  <button
    v-if="isLandscape"
    class="exit-btn"
    @click="toggleLandscape"
  >
    ✕ 退出横屏
  </button>
</template>

<style scoped>
.exit-btn {
  position: fixed;
  top: 12px;
  right: 12px;
  z-index: 1001;
}
</style>
```

# Vue Table Touch Scroll

一个专为移动端优化的 Vue 3 自定义指令，让桌面端 Table 组件在触摸设备上拥有原生般的流畅滚动体验。

## 核心特性

- 🚀 **物理仿真引擎** — 基于 Delta Time 的帧率归一化，消除 60Hz/90Hz/120Hz 屏幕的速度差异，提供一致的惯性体感
- 📱 **方向轴锁定** — 自动判定手势意图，过滤对角线抖动，每次手势只锁定在一个轴向
- 🔄 **智能边缘感知** — 滚动触达边界时自动交还控制权给浏览器，兼容 iOS 侧滑返回等原生行为
- 🛑 **急停点击保护** — 区分"用户想点击"与"用户想刹车"，仅在高速惯性滚动时拦截点击事件，防止误触
- 🖐️ **多指追踪** — 通过 Touch ID 追踪锁定活跃手指，防止多指交替导致的坐标跳变
- ⚡ **脏值检测** — 仅在滚动位置实际变化时触发渲染和事件派发，避免无效开销
- 🔧 **UI 库预设** — 内置 Element Plus、Ant Design Vue、Naive UI、VxeTable 等主流 UI 库的预设配置
- 🎯 **零依赖** — 纯原生实现，无需额外依赖

## 适用场景

- Element Plus 的 `el-table` 组件
- Ant Design Vue 的 `a-table` 组件
- Naive UI 的 `n-data-table` 组件
- VxeTable、Arco Design、PrimeVue、Vuetify 等表格组件
- 任何需要优化移动端触摸滚动的自定义表格

## 快速开始

### 全局注册

```ts [main.ts]
import { createApp } from 'vue'
import App from './App.vue'
import VueTableTouchScroll from 'vue-table-touch-scroll'

const app = createApp(App)
app.use(VueTableTouchScroll)
app.mount('#app')
```

### 在模板中使用

```vue [MyComponent.vue]
<template>
  <div v-table-touch-scroll="{ preset: 'element-plus' }">
    <el-table :data="tableData">
      <el-table-column prop="name" label="Name" />
      <el-table-column prop="age" label="Age" />
      <el-table-column prop="address" label="Address" />
    </el-table>
  </div>
</template>
```

## 解决了什么问题

在移动端直接使用桌面端 Table 组件时，通常会遇到以下问题：

- **滚动不跟手** — 原生 `overflow: auto` 在部分 UI 库的表格嵌套结构中表现不佳，触摸拖拽时响应迟钝或完全无法滚动
- **缺少惯性反馈** — 松手后没有惯性滑动，用户需要反复拖拽才能到达目标位置
- **表头滚动撕裂** — 表体滚动时表头未能同步更新，产生视觉错位
- **不同设备体感差异** — 60Hz 和 120Hz 屏幕上的惯性距离不一致，无法提供统一的交互体验
- **手势冲突** — 滚动到边界后无法触发 iOS 侧滑返回等浏览器原生行为

`v-table-touch-scroll` 通过接管触摸事件流，在指令内部实现了一套完整的滚动物理引擎来解决上述问题。以下是几个核心方案：

### 历史轨迹队列

采用滑动窗口采样替代传统的加权平均，仅保留最近 50ms 内的触摸点来计算松手速度，从根源上消除低端机采样抖动和高刷屏幕采样过密的影响：

```ts
ctx.touchTracker = ctx.touchTracker.filter((p) => now - p.time <= 50)
const oldestPoint = ctx.touchTracker[0]
ctx.velocity = -(t.clientX - oldestPoint.x) / (now - oldestPoint.time)
```

### 帧率归一化

惯性阶段的物理衰减基于 Delta Time 归一化，使得同一次滑动在 60Hz 和 120Hz 屏幕上产生相同的滚动距离：

```ts
const dt = Math.min(now - ctx.lastFrameTime, MAX_DT)
ctx.velocity *= ctx.friction ** (dt / 16.67)
```

### 同步事件派发

配合脏值检测，主动派发 `scroll` 事件强制表头即时同步，解决原生 `scroll` 异步导致的视觉撕裂：

```ts
if (el.scrollLeft !== prevX || el.scrollTop !== prevY) {
  el.dispatchEvent(new Event('scroll'))
}
```

### 边缘感知

当滚动触达容器边界时自动交还控制权，确保 iOS 侧滑返回等原生行为不被阻断：

```ts
const atLeft = el.scrollLeft <= 1 && dx > 0
const atRight = el.scrollLeft >= maxScrollLeft - 1 && dx < 0
if (atLeft || atRight) isAtEdge = true
```

**vue3-mobile-table** 是一款专为 Vue 3 生态设计的轻量化、高性能的移动端表格引擎。

它解决两大核心痛点：

- 成熟的移动端表格组件匮乏
- 成熟的桌面端组件在移动端使用时体验糟糕

通过极简的指令式接入，只需一行代码，即可赋予 Element Plus、Ant Design Vue 等任意 PC 端表格组件原生级的丝滑移动端交互体验。

## 核心特性

- **物理仿真惯性引擎** — 采用基于 Delta Time 的帧率归一化技术，彻底消除不同屏幕刷新率带来的速度差异，在任意 HZ 屏幕下均能提供一致的丝滑体感。内置智能惯性保护：识别手指停顿意图，有效防止误触发惯性滑动。
- **手势意图与方向轴锁定** — 毫秒级判定手势意图，自动过滤对角线抖动。在整个手势期间将操作锁定在单一轴向（横向或纵向），彻底解决长表格浏览过程中的「斜向漂移」痛点。
- **智能边缘感知机制** — 自动探测滚动边界。当滚动触及尽头时，平滑地将事件控制权交还给浏览器，完美兼容 iOS 侧滑返回等系统级原生交互。
- **动态急停点击保护** — 智能区分「刹车」与「触发」。仅在高速惯性滚动中拦截点击事件以实现即时避震，防止在翻阅数据时发生误触。
- **多指追踪与手势自适应** — 基于 Touch ID 锁定活跃指代，消除多指交替时的坐标跳变。检测到多指手势（如缩放）时，引擎自动进入静默模式，由浏览器接管原生行为。
- **主流 UI 库深度适配** — 内置 Element Plus、Ant Design Vue、Naive UI、VxeTable 等十余款主流组件库的预设配置，一行代码即可实现深度的交互优化。
- **多态设备自适应 (Lazy Hijacking)** — 针对 Surface/触屏笔记本等混合设备优化的状态机。纯鼠标操作零侵入，仅在触摸触发时按需唤醒，确保不同输入方式间的无缝切换。
- **横屏场景手势校准** — 深度适配 CSS `transform: rotate()` 场景。通过坐标轴智能映射自动修正触摸轨迹，确保横屏模式下的滑动方向与操作直觉完全一致，且无额外计算开销。
- **嵌套表格手势仲裁** — 支持父子表格嵌套场景（如展开行内嵌子表）。通过被动跟踪与中途边界穿透机制，确保同一时刻仅一个表格实例响应手势，子表滚动到边界后零丢帧无缝交接给父表。
- **轻量化零依赖** — 纯原生 TypeScript 实现，无任何第三方包依赖。

## 解决了什么问题

Vue 生态中尚未出现可对标主流桌面表格栈的移动端 Table 方案。在报表等复杂业务场景下，移动端同样需要具备高性能与完整交互能力的表格组件，但当前生态存在明显空缺。

实际落地通常只有两种路径：其一，自研表格组件，但触控交互、滚动机制与复杂功能实现成本极高，开发与维护负担沉重；其二，复用 Element Plus 等桌面端表格组件，但由于未针对移动端优化，在触控与小屏场景下普遍存在体验问题。

复用桌面表格时，常见痛点包括：

- **对角线滑动误触** — 用户本想横向查看表格数据，却因手指路径不直而意外触发页面纵向滚动，导致视线丢失
- **滚动方向不纯粹** — 手指轻微倾斜就可能在横纵轴之间频繁切换，滑动路径显得杂乱
- **滚动不跟手** — 在复杂 UI 组件嵌套场景中，原生 `overflow: auto` 常出现明显滞后
- **缺少惯性反馈** — 松手后滚动立即中断或惯性生硬，远距离浏览需要反复拖拽
- **表头滚动撕裂** — 表体滚动时固定表头无法像素级同步，出现错位或延迟
- **不同设备体感差异** — 刷新率差异导致同一套代码在不同设备上手感不一致
- **手势冲突** — 多指操作坐标跳变，或边界处干扰 iOS 侧滑返回等系统手势
- **混合设备干扰** — 在 Surface、触屏笔记本等设备上盲目劫持会影响鼠标与触控板使用
- **CSS 旋转横屏坐标错乱** — `transform: rotate()` 场景下触摸物理方向与逻辑滚动方向错位

`v-mobile-table` 通过接管触摸事件流，在指令内以滚动物理引擎应对上述问题；实现细节见 [工作原理](/zh/how-it-works/how-it-works)。

## 工作原理

该指令通过接管触摸事件流，在内部实现方向判定、速度采样、惯性衰减、边缘交还与混合设备策略等完整滚动物理机制，保证移动端滚动既跟手又稳定。

你可以在 [工作原理](/zh/how-it-works/how-it-works) 中查看完整机制说明与关键实现思路。

## 适用场景

### 数据查看

面向移动端的宽表、财务报表、对账清单等复杂数据展示场景。需要在有限屏幕内支持横向拖拽、纵向浏览、排序及嵌套表等多维交互，对表格功能、触控体验与渲染性能要求较高。

### 跨端一致体验

同一业务页面需同时覆盖 PC 与移动端，在不重写表格业务逻辑的前提下，实现一致且可预期的交互体验。

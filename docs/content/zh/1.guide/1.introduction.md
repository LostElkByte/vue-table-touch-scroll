vue-table-touch-scroll 是一个轻量级解决方案，专门用于弥合桌面表格组件与移动端触摸交互之间的体验差距。通过一个简单的指令，它可以让任何 Vue PC 表格组件在移动设备上也能获得接近原生级、丝滑流畅的滚动与交互体验。该方案开箱即用地支持主流 UI 库中的表格组件，例如：Element Plus、Ant Design Vue、Naive UI，让这些桌面端表格在移动端也能实现顺畅的触摸滚动体验。

## 核心特性

- **物理仿真引擎** — 基于 Delta Time 的帧率归一化，消除任意刷新率屏幕之间的速度差异，提供一致的惯性体感。内置惯性保护：手指停顿超过 80ms 再松开时自动清零速度，防止误触发惯性滚动
- **方向轴锁定** — 在每次触摸开始时自动判定手势意图，过滤对角线抖动，整个手势期间只锁定在一个轴向（横向或纵向）
- **智能边缘感知** — 滚动触达边界时自动交还控制权给浏览器，兼容 iOS 侧滑返回等原生行为
- **急停点击保护** — 区分"用户想点击"与"用户想刹车"，仅在高速惯性滚动时拦截点击事件，防止误触
- **多指追踪与锁定** — 通过 Touch ID 追踪锁定活跃手指，防止多指交替导致的坐标跳变。检测到多指时引擎完全静默，将控制权交还浏览器处理原生手势（如缩放）
- **UI 库预设** — 内置 Element Plus、Ant Design Vue、Naive UI、VxeTable、Arco Design、PrimeVue、Vuetify 等主流 UI 库的预设配置
- **混合设备懒接管** — 四态状态机（dormant/standby/pending-active/active）：纯 PC 零侵入，混合设备（Surface / 触屏笔记本）按需劫持，通过 pending-active 中间态消除误触激活
- **CSS 旋转坐标逆变换** — 支持 CSS `transform: rotate()` 横屏场景，通过离散角度轴交换自动矫正触摸坐标，零三角函数开销
- **零依赖** — 纯原生实现，无需额外依赖

## 解决了什么问题

Vue 生态中尚未出现可对标主流桌面表格栈的移动端 Table 方案。在报表等复杂业务场景下，移动端同样需要具备高性能与完整交互能力的表格组件，但当前生态存在明显空缺。

实际落地通常只有两种路径：其一，自研表格组件，但触控交互、滚动机制与复杂功能实现成本极高，开发与维护负担沉重；其二，复用 Element Plus 等桌面端表格组件，但由于未针对移动端优化，在触控与小屏场景下普遍存在体验问题。

复用桌面表格时，常见痛点包括：

- **对角线滑动误触** — 用户本想横向查看表格数据，却因手指路径不直而意外触发页面纵向滚动，导致视线丢失
- **滚动方向不纯粹** — 手指轻微倾斜就可能在横纵轴之间频繁切换，滑动路径显得杂乱
- **滚动不跟手** — 在复杂 UI 组件嵌套场景中，原生 `overflow: auto` 常出现明显滞后
- **缺少惯性反馈** — 松手后滚动立即中断或惯性生硬，远距离浏览需要反复拖拽
- **表头滚动撕裂** — 表体滚动时固定表头无法像素级同步，出现错位或延迟
- **不同设备体感差异** — 刷新率差异导致同一套代码在不同设备上手感不一致
- **手势冲突** — 多指操作坐标跳变，或边界处干扰 iOS 侧滑返回等系统手势
- **混合设备干扰** — 在 Surface、触屏笔记本等设备上盲目劫持会影响鼠标与触控板使用
- **CSS 旋转横屏坐标错乱** — `transform: rotate()` 场景下触摸物理方向与逻辑滚动方向错位

`v-table-touch-scroll` 通过接管触摸事件流，在指令内以滚动物理引擎应对上述问题；实现细节见 [工作原理](/zh/guide/how-it-works)。

## 工作原理

该指令通过接管触摸事件流，在内部实现方向判定、速度采样、惯性衰减、边缘交还与混合设备策略等完整滚动物理机制，保证移动端滚动既跟手又稳定。

你可以在 [工作原理](/zh/guide/how-it-works) 中查看完整机制说明与关键实现思路。

## 适用场景

### 数据查看

面向移动端的宽表、财务报表、对账清单等复杂数据展示场景。需要在有限屏幕内支持横向拖拽、纵向浏览、排序及嵌套表等多维交互，对表格功能、触控体验与渲染性能要求较高。

### 跨端一致体验

同一业务页面需同时覆盖 PC 与移动端，在不重写表格业务逻辑的前提下，实现一致且可预期的交互体验。

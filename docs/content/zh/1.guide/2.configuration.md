本页介绍 `v-mobile-table` 指令的常见用法。完整的配置项和类型定义请参考 [API](/zh/api/parameters)。

## 安装

::code-group

```bash [npm]
npm install vue3-mobile-table
```

```bash [yarn]
yarn add vue3-mobile-table
```

```bash [pnpm]
pnpm add vue3-mobile-table
```

::

## 全局注册

```ts [main.ts]
import { createApp } from 'vue'
import App from './App.vue'
import Vue3MobileTable from 'vue3-mobile-table'

const app = createApp(App)
app.use(Vue3MobileTable)
app.mount('#app')
```

## 局部注册

如果不需要全局注册，可以在组件中直接导入指令：

```vue
<script setup lang="ts">
import { vMobileTable } from 'vue3-mobile-table'
</script>

<template>
  <div v-mobile-table="{ preset: 'element-plus' }">
    <el-table :data="tableData" />
  </div>
</template>
```


## 在模板中使用

```vue [MyComponent.vue]
<template>
  <div v-mobile-table="{ preset: 'element-plus' }">
    <el-table :data="tableData">
      <el-table-column prop="name" label="Name" />
      <el-table-column prop="age" label="Age" />
      <el-table-column prop="address" label="Address" />
    </el-table>
  </div>
</template>
```

## 使用 UI 库预设

对于已内置预设的 UI 库，直接传入 `preset` 即可，无需手动查找滚动容器选择器：

```vue
<template>
  <!-- Element Plus -->
  <div v-mobile-table="{ preset: 'element-plus' }">
    <el-table :data="tableData">
      <el-table-column prop="name" label="Name" />
    </el-table>
  </div>

  <!-- Ant Design Vue -->
  <div v-mobile-table="{ preset: 'ant-design-vue' }">
    <a-table :data-source="dataSource" :columns="columns" />
  </div>

  <!-- VxeTable -->
  <div v-mobile-table="{ preset: 'vxe-table' }">
    <vxe-table :data="tableData">
      <vxe-column field="name" title="Name" />
    </vxe-table>
  </div>
</template>
```

::callout{icon="lucide:info"}
支持的预设列表和对应选择器请查看 [API — TablePreset](/zh/api/parameters#tablepreset)。
::

## 使用自定义选择器

对于没有内置预设的 UI 库，通过 `selector` 指定表格内部的滚动容器：

```vue
<template>
  <div v-mobile-table="{ selector: '.my-scroll-container' }">
    <MyTable />
  </div>
</template>
```

::callout{icon="lucide:triangle-alert" color="amber"}
`selector` 的优先级高于 `preset`。如果两者同时提供，将使用 `selector` 指定的元素。如果都未提供，则默认使用绑定指令的元素本身。
::

## 自定义物理参数

根据使用场景调整摩擦力和拖拽阈值：

```vue
<template>
  <!-- 更丝滑：高摩擦力（衰减慢），低阈值 -->
  <div v-mobile-table="{ preset: 'element-plus', friction: 0.98, dragThreshold: 3 }">
    <el-table :data="tableData" />
  </div>

  <!-- 更精确：低摩擦力（衰减快），高阈值 -->
  <div v-mobile-table="{ preset: 'element-plus', friction: 0.85, dragThreshold: 10 }">
    <el-table :data="tableData" />
  </div>
</template>
```

## 动态控制

传入 `false` 可完全禁用指令——所有事件监听和样式劫持都会被彻底清理：

```vue
<template>
  <div v-mobile-table="directiveEnabled ? { preset: 'element-plus' } : false">
    <el-table :data="tableData" />
  </div>
</template>

<script setup lang="ts">
import { ref } from 'vue'

const directiveEnabled = ref(true)
</script>
```

也可以通过 `enabled` 属性控制，效果完全一致：

```vue
<template>
  <div v-mobile-table="{ enabled: directiveEnabled, preset: 'element-plus' }">
    <el-table :data="tableData" />
  </div>
</template>
```

::callout{icon="lucide:info"}
`enabled: false` 与传 `false` 等价——两者都会完全销毁指令实例（移除所有事件监听并还原原始样式）。你**不需要**用它来做设备判断，指令的 `mode: 'auto'` 已经自动处理了设备检测。
::

## 设备检测模式

默认情况下，指令使用 `mode: 'auto'` 自动检测设备类型并采用最优策略：

- **纯 PC**：休眠状态 — 不绑定任何事件，零开销
- **纯移动端（手机/平板）**：立即激活完整触摸处理
- **混合设备（Surface / 触屏笔记本）**：懒接管 — 仅在确认真实触摸拖拽后才激活，避免对鼠标/触控板用户造成干扰

使用 `mode: 'always'` 可强制启用触摸处理，忽略设备类型。适用于调试或明确知道目标环境的场景：

```vue
<template>
  <div v-mobile-table="{ preset: 'element-plus', mode: 'always' }">
    <el-table :data="tableData" />
  </div>
</template>
```

::callout{icon="lucide:info"}
大多数场景下，推荐使用 `mode: 'auto'`（默认值）。仅在有明确需求绕过设备检测时才使用 `'always'`。
::

## 禁用边缘检测

默认情况下，指令会在滚动触达容器边界时将控制权交还浏览器，以支持 iOS 侧滑返回等原生手势。在全屏覆盖层场景（如横屏模式）下，这个行为是不需要的。设置 `disableEdgeDetection: true` 可保持指令的完全接管：

```vue
<template>
  <div v-mobile-table="{ preset: 'element-plus', disableEdgeDetection: true }">
    <el-table :data="tableData" />
  </div>
</template>
```

::callout{icon="lucide:triangle-alert" color="amber"}
仅在表格处于全屏覆盖层或需要阻止浏览器原生边缘手势的场景下启用此选项。在常规页面布局中，边缘检测能提升用户体验（允许侧滑返回和外层滚动）。
::

## CSS 旋转支持

当通过 CSS `transform: rotate()` 模拟移动端横屏时，触摸坐标轴与元素逻辑轴不再对齐。`rotation` 参数会告知指令当前旋转角度，自动执行坐标逆变换：

```vue
<template>
  <div
    v-mobile-table="{ preset: 'element-plus', rotation: 90 }"
    class="landscape-container"
  >
    <el-table :data="tableData" />
  </div>
</template>

<style scoped>
.landscape-container {
  transform: rotate(90deg);
  transform-origin: top left;
  width: 100vh;
  height: 100vw;
  translate: 100vw 0;
}
</style>
```

支持的值：`0`（默认）、`90`、`-90`、`180`。完整的横屏切换示例请查看[移动端横屏方案](/zh/landscape/landscape)。

## 生命周期回调

```vue
<script setup lang="ts">
const scrollOptions = {
  preset: 'element-plus' as const,
  onScrollStart: () => {
    // 滚动开始：关闭浮层、记录开始时间等
  },
  onScrollEnd: () => {
    // 滚动结束：触发埋点、加载更多数据等
  },
}
</script>

<template>
  <div v-mobile-table="scrollOptions">
    <el-table :data="tableData" />
  </div>
</template>
```

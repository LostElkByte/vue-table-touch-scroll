## 核心工作原理

`v-table-touch-scroll` 通过接管触摸事件流，在指令内部实现方向判定、速度采样、惯性衰减、边缘交还与设备策略，构建一套完整的滚动物理引擎，让桌面表格在移动端依然保持稳定、跟手、可预期的交互体验。

### 方向轴锁定

当手指位移超过阈值时自动判定手势意图，整个手势期间锁定在单一轴向：

```ts
if (!ctx.gestureDirection) {
  if (Math.abs(dx) > threshold || Math.abs(dy) > threshold) {
    ctx.gestureDirection = Math.abs(dx) > Math.abs(dy) ? 'horizontal' : 'vertical'
    ctx.lockedDirection = ctx.gestureDirection
  }
}
```

### 历史轨迹队列

采用滑动窗口采样替代传统的加权平均，仅保留最近 50ms 内的触摸点来计算松手速度，从根源上消除低端机采样抖动和高刷屏幕采样过密的影响：

```ts
ctx.touchTracker.push({ time: now, x, y })
ctx.touchTracker = ctx.touchTracker.filter((p) => now - p.time <= 50)

const oldest = ctx.touchTracker[0]
const dt = Math.max(1, now - oldest.time)
if (ctx.gestureDirection === 'horizontal') {
  ctx.velocity = -(x - oldest.x) / dt
} else {
  ctx.velocity = -(y - oldest.y) / dt
}
```

### 帧率归一化

惯性阶段的物理衰减基于 Delta Time 归一化，使得同一次滑动在任意刷新率的屏幕上都产生相同的滚动距离：

```ts
const rawDt = ctx.lastFrameTime ? now - ctx.lastFrameTime : 16.67
const dt = Math.min(rawDt, MAX_DT)
ctx.velocity *= ctx.friction ** (dt / 16.67)
```

### 同步事件派发

配合脏值检测，主动派发 `scroll` 事件强制表头即时同步，解决原生 `scroll` 异步导致的视觉撕裂：

```ts
if (el.scrollLeft !== prevX || el.scrollTop !== prevY) {
  el.dispatchEvent(new Event('scroll'))
}
```

### 边缘感知

当滚动触达容器边界时自动交还控制权，同时覆盖横向和纵向检测以及无需滚动的场景：

```ts
if (isHorizontal) {
  const maxScroll = el.scrollWidth - el.clientWidth
  const atLeft = el.scrollLeft <= 1 && dx > 0
  const atRight = el.scrollLeft >= maxScroll - 1 && dx < 0
  const noScroll = el.scrollWidth <= el.clientWidth
  if (atLeft || atRight || noScroll) isAtEdge = true
}
```

### 混合设备懒接管

通过四态状态机处理混合环境 — 纯 PC 保持休眠，移动端立即激活，混合设备使用哨兵事件，仅在确认真实触摸拖拽后才触发完整劫持：

```ts
// standby：仅哨兵事件，不修改样式
// pending-active：touchstart 已记录，但样式未劫持
// active：overflow: hidden 已应用，绑定全套触摸事件
if (dx > ctx.dragThreshold || dy > ctx.dragThreshold) {
  activateHijacking(ctx)  // standby → active
}
```

## 问题背景

- **对角线滑动误触** — 用户本想横向查看表格数据，却因手指路径不直而意外触发页面纵向滚动，导致视线丢失
- **滚动方向不纯粹** — 手指轻微倾斜就可能在横纵轴之间频繁切换，滑动路径显得杂乱
- **滚动不跟手** — 在复杂 UI 组件嵌套场景中，原生 `overflow: auto` 常出现明显滞后
- **缺少惯性反馈** — 松手后滚动立即中断或惯性生硬，远距离浏览需要反复拖拽
- **表头滚动撕裂** — 表体滚动时固定表头无法像素级同步，出现错位或延迟
- **不同设备体感差异** — 刷新率差异导致同一套代码在不同设备上手感不一致
- **手势冲突** — 多指操作坐标跳变，或边界处干扰 iOS 侧滑返回等系统手势
- **混合设备干扰** — 在 Surface、触屏笔记本等设备上盲目劫持会影响鼠标与触控板使用
- **CSS 旋转横屏坐标错乱** — `transform: rotate()` 场景下触摸物理方向与逻辑滚动方向错位

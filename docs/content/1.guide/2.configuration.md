# 配置指南

Vue Table Touch Scroll 提供了灵活的配置选项，支持通过 UI 库预设或自定义选择器来定位滚动容器。

## 配置选项

### TableTouchScrollOptions

| 属性 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `enabled` | `boolean` | `true` | 是否启用自定义滚动 |
| `threshold` | `number` | `5` | 判定滚动方向前的位移阈值（像素），用于区分轻微抖动点击和有意识的拖拽 |
| `preset` | `TablePreset` | - | UI 库预设名称，自动应用对应 UI 库的滚动容器选择器 |
| `selector` | `string` | - | 目标滚动元素的 CSS 选择器，优先级高于 `preset` |

### TablePreset

支持的 UI 库预设：

| 预设值 | 对应 UI 库 | 选择器 |
|--------|-----------|--------|
| `'element-plus'` | Element Plus | `.el-scrollbar__wrap` |
| `'ant-design-vue'` | Ant Design Vue | `.ant-table-body` |
| `'arco-design'` | Arco Design | `.arco-table-body` |
| `'naive-ui'` | Naive UI | `.n-scrollbar-container` |
| `'primevue'` | PrimeVue | `.p-datatable-table-container` |
| `'vuetify'` | Vuetify | `.v-table__wrapper tbody` |
| `'vxe-table'` | VxeTable | `.vxe-table--body-inner-wrapper` |

## 使用方式

### 方式一：使用 UI 库预设（推荐）

对于支持的 UI 库，直接使用预设名称即可：

```vue
<template>
  <!-- Element Plus -->
  <div v-table-touch-scroll="{ preset: 'element-plus' }">
    <el-table :data="tableData">
      <el-table-column prop="name" label="Name" />
    </el-table>
  </div>

  <!-- Ant Design Vue -->
  <div v-table-touch-scroll="{ preset: 'ant-design-vue' }">
    <a-table :dataSource="dataSource" :columns="columns" />
  </div>

  <!-- Naive UI -->
  <div v-table-touch-scroll="{ preset: 'naive-ui' }">
    <n-data-table :data="data" :columns="columns" />
  </div>
</template>
```

### 方式二：使用自定义选择器

对于不支持的 UI 库或自定义组件，可以使用 CSS 选择器：

```vue
<template>
  <div v-table-touch-scroll="{ selector: '.my-custom-scroll-container' }">
    <CustomTable>
      <div class="my-custom-scroll-container">
        <!-- 表格内容 -->
      </div>
    </CustomTable>
  </div>
</template>
```

### 方式三：调整阈值

如果你的表格需要更灵敏或更严格的响应，可以调整阈值：

```vue
<template>
  <!-- 更灵敏：较小的阈值，轻微移动即触发滚动 -->
  <div v-table-touch-scroll="{ preset: 'element-plus', threshold: 3 }">
    <el-table :data="tableData" />
  </div>

  <!-- 更严格：较大的阈值，需要明显移动才触发滚动 -->
  <div v-table-touch-scroll="{ preset: 'element-plus', threshold: 10 }">
    <el-table :data="tableData" />
  </div>
</template>
```

### 方式四：动态启用/禁用

可以根据条件动态启用或禁用指令：

```vue
<template>
  <div v-table-touch-scroll="{ enabled: isMobile, preset: 'element-plus' }">
    <el-table :data="tableData" />
  </div>
</template>

<script setup lang="ts">
import { ref } from 'vue'

const isMobile = ref(true)

// 根据设备类型切换
const checkDevice = () => {
  isMobile.value = /Android|webOS|iPhone|iPad|iPod|BlackBerry|IEMobile|Opera Mini/i.test(
    navigator.userAgent
  )
}
</script>
```

## 局部注册

如果你不想全局注册，可以在单个组件中局部使用：

```vue
<script setup lang="ts">
import { vTableTouchScroll } from 'vue-table-touch-scroll'
</script>

<template>
  <div v-table-touch-scroll="{ preset: 'element-plus' }">
    <el-table :data="tableData" />
  </div>
</template>
```

## TypeScript 支持

库内置了完整的 TypeScript 类型定义：

```ts
import type { TableTouchScrollOptions, TablePreset } from 'vue-table-touch-scroll'

const options: TableTouchScrollOptions = {
  enabled: true,
  threshold: 5,
  preset: 'element-plus'
}

const presetValue: TablePreset = 'ant-design-vue'
```

## 优先级规则

配置优先级从高到低：

1. `selector` - 如果提供了自定义选择器，优先使用
2. `preset` - 如果提供了预设，使用对应的选择器
3. 默认行为 - 如果都未提供，滚动容器为绑定指令的元素本身

```vue
<template>
  <!-- selector 优先级高于 preset -->
  <div v-table-touch-scroll="{ 
    selector: '.custom-container',  // 将使用这个
    preset: 'element-plus'       // 这个会被忽略
  }">
    <el-table :data="tableData" />
  </div>
</template>
```

## 最佳实践

1. **优先使用预设**：对于支持的 UI 库，使用预设可以避免手动查找选择器
2. **合理设置阈值**：根据表格内容和使用场景调整阈值
3. **移动端检测**：可以结合用户代理检测，在桌面端禁用指令
4. **嵌套使用**：指令可以嵌套使用，但要注意避免冲突

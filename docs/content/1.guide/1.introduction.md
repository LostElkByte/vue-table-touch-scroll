vue-table-touch-scroll is a lightweight solution dedicated to bridging the gap between desktop tables and mobile touch interactions. With a simple directive, it ensures that any Vue PC table component delivers a native-grade, silky-smooth scrolling and interaction experience on mobile devices. This includes seamless out-of-the-box support for table components in mainstream UI libraries like Element Plus, Ant Design Vue, and Naive UI.

## Core Features

- **Physics Simulation Engine** — Frame-rate normalized via Delta Time, eliminating speed differences across any screen refresh rate for consistent inertia feel. Includes inertia protection: if the finger pauses for more than 80ms before release, velocity is automatically cleared to prevent unintended scrolling
- **Axis Locking** — Automatically determines gesture intent at the start of each touch, filters diagonal jitter, and locks the gesture to a single axis (horizontal or vertical) for the entire duration
- **Smart Edge Detection** — Automatically yields control back to the browser when scrolling reaches boundaries, compatible with iOS swipe-back and other native behaviors
- **Emergency Stop Click Protection** — Distinguishes between "user wants to click" and "user wants to brake". Only blocks click events during high-speed inertia scrolling to prevent accidental taps
- **Multi-finger Tracking & Lock** — Locks the active finger via Touch ID to prevent coordinate jumps during multi-finger alternation. When multiple fingers are detected, the engine goes completely silent and hands control to the browser for native gestures (pinch-to-zoom, etc.)
- **UI Library Presets** — Built-in preset configurations for Element Plus, Ant Design Vue, Naive UI, VxeTable, Arco Design, PrimeVue, and Vuetify
- **Hybrid Device Lazy Hijacking** — Four-state machine (dormant/standby/pending-active/active): zero intrusion on pure PC, on-demand hijacking for hybrid devices (Surface / touchscreen laptops), with a pending-active intermediate state to eliminate accidental activation
- **CSS Rotation Coordinate Transform** — Supports CSS `transform: rotate()` landscape scenarios by applying inverse coordinate transforms via discrete angle axis-swapping, with zero trigonometric overhead
- **Zero Dependencies** — Pure native implementation, no extra dependencies needed

## How It Works

The directive intercepts touch event flow and runs a complete in-directive scroll physics pipeline, including axis decision, velocity sampling, inertia decay, edge yielding, and hybrid-device strategies for stable mobile behavior.

Read [How It Works](/guide/how-it-works) for the full mechanism breakdown and implementation details.

## Use Cases

- **Data viewing**: mobile scenarios with wide tables, financial reports, and reconciliation sheets where horizontal dragging, vertical browsing, sorting, and nested tables coexist
- **Cross-platform consistency**: one business page serving both desktop and mobile, with predictable and maintainable touch interaction without rewriting table business logic

# Troubleshooting

This guide covers common issues you might encounter when using Vue Table Touch Scroll and how to resolve them.

## Common Issues

### Directive Not Working

**Problem**: The directive is applied but touch scrolling doesn't work.

**Solutions**:

1. **Check if the directive is registered**:
   ```ts
   // Make sure you've registered the directive
   app.use(VueTableTouchScroll)
   ```

2. **Verify the table element exists**:
   The directive needs to be applied to a valid table element. Make sure your table component renders a `<table>` element.

3. **Check browser compatibility**:
   Ensure you're testing on a device or browser that supports touch events.

### Scrolling is Choppy or Laggy

**Problem**: Touch scrolling works but feels sluggish or choppy.

**Solutions**:

1. **Adjust scroll speed**:
   ```vue
   <el-table v-table-touch-scroll="{ scrollSpeed: 1.2 }" :data="data">
   ```

2. **Reduce table complexity**:
   - Limit the number of visible rows using pagination
   - Simplify column rendering logic
   - Use virtual scrolling for large datasets

3. **Disable momentum scrolling**:
   ```vue
   <el-table v-table-touch-scroll="{ enableMomentum: false }" :data="data">
   ```

### Conflicts with UI Library Scrolling

**Problem**: The directive conflicts with the UI library's built-in scrolling.

**Solutions**:

1. **Disable library's native scrolling**:
   ```vue
   <!-- For Element Plus -->
   <el-table v-table-touch-scroll :data="data" :scroll-behavior="'none'">
   ```

2. **Adjust threshold**:
   ```vue
   <el-table v-table-touch-scroll="{ threshold: 15 }" :data="data">
   ```

### TypeScript Errors

**Problem**: TypeScript shows errors when using the directive.

**Solutions**:

1. **Add type declarations**:
   ```ts
   // env.d.ts or global.d.ts
   import 'vue-table-touch-scroll'
   ```

2. **Use explicit typing**:
   ```vue
   <script setup lang="ts">
   import type { TableTouchScrollOptions } from 'vue-table-touch-scroll'
   
   const options: TableTouchScrollOptions = {
     scrollSpeed: 1.5
   }
   </script>
   ```

### Not Working on Desktop

**Problem**: The directive only works on mobile, not on desktop.

**Explanation**: This is expected behavior. Vue Table Touch Scroll is designed specifically for touch interactions on mobile devices. Desktop users can use the standard mouse scrolling.

If you need to enable it for desktop testing:

```vue
<el-table v-table-touch-scroll="{ enableDesktop: true }" :data="data">
```

## Performance Optimization

### Large Tables

For tables with many rows or columns:

1. **Use pagination**:
   ```vue
   <el-table 
     v-table-touch-scroll 
     :data="paginatedData"
     :page-size="20"
   >
   ```

2. **Implement virtual scrolling**:
   Consider using a virtual scrolling library for extremely large datasets.

3. **Optimize rendering**:
   - Use `v-memo` for static content
   - Avoid complex computed properties in cells
   - Lazy load images and heavy content

### Memory Leaks

If you notice memory issues:

1. **Ensure proper cleanup**:
   The directive automatically cleans up event listeners when the component is unmounted. If you're manually managing instances, make sure to clean up properly.

2. **Check for circular references**:
   Avoid creating circular references in your table data.

## Getting Help

If you're still experiencing issues:

1. **Check the GitHub Issues**: [Vue Table Touch Scroll Issues](https://github.com/LostElkByte/vue-table-touch-scroll/issues)
2. **Create a minimal reproduction**: Use a tool like StackBlitz or CodeSandbox
3. **Open a new issue**: Provide details about your environment, Vue version, and UI library

## Reporting Bugs

When reporting a bug, please include:

- Vue version
- UI library and version (Element Plus, Ant Design Vue, etc.)
- Browser and device information
- Minimal code reproduction
- Expected vs actual behavior
- Console errors (if any)

Visit our [GitHub repository](https://github.com/LostElkByte/vue-table-touch-scroll) to report issues.
